# API-Health-Check-Monitor

Objective: Build a system to monitor REST API endpoints for availability, response time, and correctness.

Features:

• Endpoint Monitoring: Check multiple API endpoints simultaneously (identify at least 3 publicly available APIs for monitoring)

• Response Validation: Verify response structure and content

• Alert System: Send notifications for failures

• Dashboard: Display uptime statistics and response times

• Historical Logging: Store monitoring data for analysis.

Tech: yaml configuration file.
